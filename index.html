Climate

Notice! To fully complete this module you will have neededd to complete the RPi and Control Panel module!

Step 1.

Firstly, we will need to create a directory to work in. To do this use the following command "sudo mkdir temp" into the console.

Step 2.

Enter the command "cd temp" to go to our newly created folder.

Step 3.

We will need to download the necessary files to use our climate reading chip. To download this, enter the following command "sudo wget -O bme280.py http://bit.ly/bme280py" into the console.

Step 4.

Additionally, we need to download the necessary python libraries in order to read the output from the chip and properly use its functions. To do this, enter the following command "sudo apt-get install -y python-smbus" into the console. If prompted, enter "y" into the console to continue with the download.

Step 5.

The following package we are going to download will allow to test if our climate reading chip has successfully been installed and is detectable by our Raspberry Pi. Enter the following command "sudo apt-get install -y i2c-tools" into the console. If prompted enter "y" into the console. Once this has completed downloading, reboot your device by entering "sudo reboot now" into the console.

Step 6.

Once your Raspberry Pi has rebooted (Step 5). Enter the following command "sudo raspi-config" into the console This will open the config menu for the Raspberry Pi.

Step 7.

Using the arrow keys, navigate to "5. Interfacing Options" and press enter when it becomes highlighted. This can be seen in the screenshot below.

Step 8.

Using the arrow keys, navigate to "P5 I2C" and press enter, as shown in the screenshot below.

Step 9.

After having completed Step 8, you will be prompted with a message asking if you would like to enable the "ARM I2C Interface". Navigate to "Yes" and hit enter. After you have done this, exit out the config menu and reboot the Raspberry Pi, if not prompted to do so.

Step 10.

Now that we have downloaded and installed all the neccessarry files/tools, it's time to test if our climate reading chip is readable by the Raspberry Pi. To do this, enter the following command "i2cdetect -y 1" into the console. If the output resembles the screenshot below, then the climate reading chip has successfully been implemented. If nothing is being detected, you should double check

Step 11.

Finally we can test the climate reading chip. To do this enter the following command "sudo python bme280.py" which can be seen in the screenshot below.

Step 12.

If the installation and setup of the climate reading chip has been successful, the output should be similar to the screenshot below.

Camera & Video

Notice! To fully complete this module you will have needed to complete the RPi and database module!

Step 1.

Navigate to the console window on your Raspberry Pi, and enter "sudo raspi-config" as shown in the screenshot below"


Step 2.

You will be presented with an onscreen menu as shown below in the screenshot. Using the arrow keys navigate to "5 Interfacing Options" and hit enter


Step 3.

You will be presented with another onscreen menu. Navigate to "P1 Camera" and press enter. This can be seen in the screenshot below


Step 4.

Navigate to "Yes" and press enter.


Step 5.

The following screen as shown in the screenshot below, will prompt you to reboot your machine. Navigate to "Yes" and press enter.


Step 6.

Now it is time to test the camera. As per the following screenshot, enter the commmand "raspistill -v -o test.jpg" into the console.


Step 7.

Once you have issued the previous command. Type "ls" into the console. If you can see a file called "test.jpg" in your current directory. Then you have successfully set-up the camera


Step 8.

Now that your camera is able to take pictures. The next step is to give it the ability to snap a video clip anytime motion is detected. To do this, we will need to download and install "motion". This will allow to complete this.


Step 9.

For the following steps, we will need to know a couple of details about our camera. As per the following screenshot, enter the command "v412-ctl -V" into the console. The information we are looking for is 'Width/Height'. In this example the Width and Height is 1024 and 768.


Step 10.

It's now time to change the configuration file for 'motion'. Enter the following command "sudo nano /etc/motion/motion.conf" into the console.


Step 11.

The first thing we have to change in the 'motion.conf' file is the deamon value. In the screenshot below we can see the deamon has been set to off. Change this value to 'on' so it says 'deamon on'. This will allow us to run the motion capture software in the background.


Step 12.

In the following screenshot, we can see the value of "stream_localhost" has been to set to "on". We need to change this value to "off". By changing this value to "off", we can now access the livestream of our camera through the internet. If we didn't change this value, we would only be able to access our camere livestream on our local network i.e. home WiFi.


Step 13.

In this step, we are upping the maximum framerate of the camera livestream. By default the framerate is set to "1" which can be seen in the screenshot below, this would give us a very choppy video. Change the frame rate to "30". Although, we changed the livestream framrate to 30, it depends on the wifi signal so this may actually be lower on the livestream. Additionally, our camera max framerate is 30, so that's why the config file was set to 30.


Step 14.

This step is important as it sets the location of where our captured motion video clips will be saved. As per the following screenshot, we can see the files are being saved to a folder located at "/var/lib/motion". To make the files easier to find, we are going to change where the video clips are being saved. Change the value "/var/lib/motion" to "/home/pi/Monitor". This will store the video clips in a folder called "Monitor" located at home on the Raspberry Pi.


Step 15.

The value of v4l2_palette is currently set to "17". Change this value to "15".


Step 16.

In Step 9 we found out the width and height of our camera. In the screenshot below, we can see the current width of the camera is set to "320". Change this value to the width of your camera, in our example, the width is 1024.


Step 17.

This step is similar to Step 16. Change the value of "240" to the height of the camera, in our case, 240. It's important to set the width and height of camera as it will give a bigger and clearer image.


Step 18.

This step is if you want to capture video seperate from the livestream. The default framerate is set to "2', change to this our cameras framerate which is 30.


Step 19.

We nearly have a working livestream of our camera. In step 11, we enabled the motion capture software to run in the background. Now we have to actually give it the permission to run. Enter "sudo nano /etc/default/motion" into the console, as shown in the screenshot below.

Step 20.

Change the value of "start_motion_deamon" to "yes"


Step 21.

In Step 14, we pointed at where to save the capture video files from the camera. Now we have to create that location. As shown in the screenshot below, enter the command "mkdir /home/pi/Monitor" into console.


Step 22.

In this step, we are simply giving 'motion' owenership of our newly created folder "Monitor".


Step 23.

Now we are giving 'motion' the ability to read and write to the folder 'Monitor'


Step 24.

Finally, we can start our livestream. Enter the following command "sudo service motion start" into the console, as shown in the screenshot below.
